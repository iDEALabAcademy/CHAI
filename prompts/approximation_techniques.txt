Approximation techniques in computing are strategies used to simplify or reduce the computational intensity of processes, trading off some degree of accuracy or precision for reduced energy consumption or reduced resource consumption. Here's a summary of the 30 approximation techniques:

1. Loop perforation: Reduces runtime by truncating the loop condition statement, sacrificing some output accuracy for speed.
2. Precision scaling: Lowers the precision of numerical calculations to save resources while affecting minimally acceptable inaccuracies.
3. Function memoization: Caches results of expensive function calls to avoid redundant computations, speeding up execution.
4. Task skipping: Omits non-critical tasks to improve performance, particularly in scenarios where not all functions are essential for acceptable outcomes.
5. Approximate data structures: Uses structures that provide estimative results for large data queries, balancing accuracy and computational overhead.
6. Probabilistic algorithms: Relies on algorithms that incorporate randomness to provide solutions that are correct with a certain probability, optimizing speed over guaranteed accuracy.
7. Truncated computations: Stops an algorithm before its complete theoretical execution, yielding a result faster but with less precision.
8. Simplified models or algorithms: Employs less complex algorithms to reduce computation needs at the expense of some detail and accuracy.
9. Dynamic precision adjustment: Adjusts the computational precision based on current system load or requirements, optimizing for performance or energy.
10. Quantization: Reduces the range of numbers or data points to smaller, manageable sets, crucial in operations like machine learning.
11. Energy-aware computation: Modifies computational fidelity for energy efficiency, crucial in battery-operated and ecological computing.
12. Data approximation: Utilizes compressed or simplified versions of data to expedite operations, often used in processing large-scale or complex datasets.
13. Selective re-computation: Reuses existing computational results when possible, recalculating only when essential changes occur.
14. Multi-fidelity modeling: Combines multiple models of varying accuracies to strike a balance between computational expense and result accuracy.
15. Relaxed consistency models: Eases consistency constraints in distributed systems to reduce synchronization overhead, boosting performance.
16. Sparse computation: Focuses resources on significant data points, ignoring zeroes in sparse matrices for efficiency.
17. Neural network pruning: Trims insignificant neural connections to streamline models for faster processing and reduced model size.
18. Approximate join techniques: Implements faster, non-exact methods for database joins to improve query times at the cost of some precision.
19. Use of surrogate functions: Substitutes complex functions with simpler ones to decrease computational demands.
20. Adaptive algorithms: Adjusts algorithmic behavior based on input data characteristics or system conditions to optimize processing.
21. Early-exit approximation: Safely terminates computation (loop or multi-stage function) when a convergence or confidence metric falls below a threshold knob, reducing iterations and wall-clock time. Trades final accuracy for speed; applicable to iterative solvers, search algorithms, and accumulation loops. Exit uses safe control-flow `break` or `return` statements.
22. Spatial downsampling (input fidelity approximation): Reduces computational intensity by processing an image or spatial data at a coarser resolution (every n-th pixel in x and y dimensions) while maintaining the same output dimensions. Uses safe fill strategies (nearest-neighbor, linear interpolation) for skipped pixels. Trades spatial detail for energy efficiency; applicable to image processing, edge detection, and filtering operations. Knob: downsample_factor [1, 2, 4].
23. Temporal decimation (data-level approximation): Reduces computational intensity in time-series processing by decimating the time axis (processing every n-th sample) and reconstructing the full-length signal using zero-order hold (ZOH) or linear interpolation. Maintains same output dimensions (full temporal reconstruction) while reducing anchor point computations. Trades temporal fidelity for energy efficiency; applicable to activity recognition, sensor signal processing, and time-series classification. Energy model: cycles_ratio = 1/decimation_factor + 0.05. Knobs: decimation_factor [1, 10], interpolation_mode [0=ZOH, 1=linear].
24. Magnitude-only FFT (frequency-domain approximation): Reduces computational intensity in signal processing by discarding phase information from FFT spectrum and retaining only magnitude, then reconstructing via inverse FFT with deterministic phase policy (zero phase, random fixed, or carry-from-baseline). Trades phase fidelity for reduced feature dimensionality and computation. Applicable to activity recognition, audio processing, and frequency-domain feature extraction. Supports two output modes: time-domain reconstruction (maintains signal shape) or magnitude-feature extraction (outputs STFT magnitude frames for direct classification). Energy model: cycles_ratio = 0.60 for time-domain mode, 0.50 for mag-features mode. Knobs: enabled [0,1], n_fft [64,128,256], hop [n_fft/4, n_fft/2], window [hann, rect], keep [time, mag_features], phase_policy [zero, random_fixed], magnitude_transform [linear, log1p], topk_bins [None, 8, 16, 32].
25. Lazy preprocessing (preprocessing approximation): Reduces preprocessing overhead in string search algorithms by deferring table construction until first mismatch or lazily building entries for encountered characters only. Uses KMP-Boyer-Moore hybrid algorithm to guarantee forward progress without infinite loops. Trades preprocessing completeness for reduced upfront computation. Applicable to pattern matching, text search, and preprocessing-heavy string operations. Energy model: cycles_ratio = preprocessing_entries_built / total_charset_size. Knobs: lazy_preprocess_enabled [0, 1], lazy_cache_enabled [0, 1], lazy_budget [16, 256].
26. Bit-shift EWMA (arithmetic simplification): Replaces floating-point exponentially weighted moving average (EWMA) computations with integer-based bit-shift arithmetic. Instead of ewma = alpha * new_sample + (1-alpha) * ewma, uses ewma = ewma + ((new_sample - ewma) >> shift_factor), where alpha ≈ 1/(2^shift_factor). Eliminates floating-point multiplications entirely in the approximate branch, trading smoothing precision for reduced energy and cycle count. Applicable to link estimators, sensor filtering, signal smoothing, and any weighted averaging computation. Energy model: removes all FP multiply operations; shift_factor=0 is exact baseline. Knob: shift_factor [0, 4] where 0=baseline (exact FP EWMA), 1=α≈0.5, 2=α≈0.25, 3=α≈0.125, 4=α≈0.0625.
27. Nibble Lookup (computation simplification): Replaces byte-level (256-entry) or per-bit popcount with a compact 16-entry nibble (4-bit) lookup table, processing each nibble of the input word independently. A 32-bit word has 8 nibbles; at stride=1 all 8 are counted (exact, baseline-equivalent popcount). At stride>1, only every n-th nibble is counted and the result is scaled proportionally (cnt = cnt * 8 / nibbles_counted), trading accuracy for fewer table lookups and loop iterations. Reduces memory footprint (16 bytes vs 256 bytes for byte-LUT) and eliminates recursion. Applicable to bitcount, population count, Hamming weight, and parity computations. Energy model: cycles_ratio = 1/nibble_stride (fewer LUT accesses per word). Knob: nibble_stride [1, 4] Integer where 1=exact (all 8 nibbles counted), 2=half nibbles (4 lookups, scale×2), 3=third nibbles (3 lookups, scale×8/3), 4=quarter nibbles (2 lookups, scale×4).
28. Hierarchical Feature Extraction (feature-level approximation): Reduces feature-extraction compute by computing cheap tier-1 features (mean, variance, energy, zero-crossing count per channel) for every sliding window, then selectively computing expensive tier-2 features (spectral centroid, spectral bandwidth, top-k FFT magnitude bins) only for windows whose gating score exceeds a threshold. Skipped windows fill tier-2 slots with zeros, preserving fixed classifier input dimension. A full_rate knob forces a fraction of windows to always compute tier-2 for calibration. Trades feature completeness for reduced FFT operations. Applicable to activity recognition, gesture detection, and sensor-based classification. Energy model: cycles_ratio = (1 + skip_ratio * tier2_cost_ratio) / (1 + tier2_cost_ratio), where tier2_cost_ratio ≈ log2(window_size). Knobs: hfe_enabled [0, 1], hfe_gate_tau [0.0, 2.0] Real (0.0 = baseline, all tier-2 computed), hfe_full_rate [0.0, 1.0] Real (fraction of windows always getting tier-2), hfe_mode ["energy", "accuracy"].
29. Radix Variation (arithmetic simplification): Replaces expensive base-10 digit extraction (repeated / 10 and % 10 — software-emulated division on MSP430, ~150 cycles each) with power-of-two radix operations (>> 4 and & 0xF for base-16, ~1 cycle each). Also supports reducing max_digits to truncate output and skip_leading_zeros to bypass zero-valued leading digits. Applicable to integer-to-string conversion, hash/signature formatting, histogram bucket labeling, and any code that extracts decimal digits from integers. When radix_mode=10 and max_digits=original length, output is bit-identical to baseline. Energy model: cycles_ratio ≈ (shift_cost / div_cost) × (max_digits / original_digits); for radix_mode=16 on MSP430 this is ~1/150 per digit. Knobs: radix_mode [2, 16] Integer (10 = baseline exact base-10; 16 = hex; 8 = octal; 2 = binary), max_digits [1, 10] Integer (original = baseline), skip_leading_zeros [0, 1] Integer (0 = baseline).
30. Pattern Segmentation (data-level approximation): Identifies repeated, homogeneous, or low-variation regions ("segments") in an input array/signal/image row and avoids doing full computation on every element. Instead, computes at representative points (every k-th element, segment boundaries, or change-triggered anchors) and fills non-representative positions using hold-last or linear interpolation. Distinct from loop perforation: all output positions are still written; skipped positions reuse a previously computed result rather than being omitted. Applicable to loops over arrays/signals/images where adjacent elements are correlated — feature extraction, filtering, thresholding, scanning, distance checks, scoring. When segment_len=1 and policy=every_k, every element is computed exactly (baseline-equivalent). Energy model: cycles_ratio ≈ 1/segment_len + fill_overhead (~2-5 cycles per filled element). Knobs: segment_len [1, 8] Integer (1 = baseline, compute every element), change_threshold [0, 50] Integer (0 = always recompute, baseline), policy [0, 2] Integer (0=every_k baseline, 1=boundary_only, 2=change_triggered), fill_strategy [0, 1] Integer (0=hold_last, 1=linear_interp).
